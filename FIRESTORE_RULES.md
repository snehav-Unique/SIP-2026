# Firestore Security Rules

Copy these rules into Firebase Console:

Firestore Database -> Rules -> replace the existing rules -> Publish.

These rules match the app's Google Sign-In allowlist in `.env`.
Emergency password access is only a local dashboard fallback and cannot securely write to Firestore without a backend.

```js
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isAllowedDean() {
      return request.auth != null
        && request.auth.token.email in [
          "dean.studentaffairs@rvce.edu.in",
          "snehav0307@gmail.com",
          "abhinandan8667@gmail.com"
        ];
    }

    match /announcements/{id} {
      allow read: if true;
      allow create, update, delete: if isAllowedDean();
    }
  }
}
```

After publishing:

1. Open `/sipannouncements/secretlogin`.
2. Sign in with one of the allowed Google accounts.
3. Go to the dean dashboard and create the announcement.

The announcement is stored in Firestore at:

```text
announcements/{autoGeneratedId}
```

Expected fields include `title`, `description`, `date`, `time`, `location`, `category`, `author`, `hasDocument`, optional file metadata, and `createdAt`.
